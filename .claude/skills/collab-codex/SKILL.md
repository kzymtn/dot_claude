# collab-codex

Plan モードで設計の判断を行う際に Codex MCP を通じてセカンドオピニオンを取得するスキル。

## トリガー条件

以下のいずれかに該当するとき自動的に発動する:

- Plan フェーズの **Phase 2: Design** — 実装アプローチを設計しているとき
- Plan フェーズの **Phase 4: Final Plan** — プランファイルを確定する前
- 複数のアーキテクチャ候補で迷っているとき
- ユーザーが「Codex に聞いて」「collab-codex」と明示したとき

## 手順

### Step 1: Codex にプロンプトを送る

`mcp__codex__codex` ツールを呼び出す。`prompt` には以下を含める:

```
# Task
<ユーザーの要求を1〜3文で要約>

# Context
<関係するファイル・コードの概要>

# Candidate Approaches
<検討中の実装案を箇条書き>

# Question
どのアプローチが最適か、理由とトレードオフを教えてください。
```

### Step 2: レスポンスを受け取る

`mcp__codex__codex-reply` ツールを呼び出してレスポンスを取得する。

### Step 3: プランへの反映

Codex の見解をプランファイルの該当セクションに組み込む。
賛同点はそのまま採用し、相違点はユーザーに提示して判断を仰ぐ。

## 注意

- Codex の意見は参考情報。最終判断はユーザーに委ねる。
- MCP が利用不可のときはスキップしてその旨をユーザーに伝える。
