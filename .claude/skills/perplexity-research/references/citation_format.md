# Citation format

## 基本原則

事実主張を含む全文に引用マーカーを付ける。
引用マーカー形式: Claude Code の場合、URLをインライン `[N]` 形式で付け、末尾に参照リストを置く。

例:
```
OpenAI の o3 モデルは 2024 年 12 月にリリースされた [1]。
...

**参照**
[1] https://openai.com/... — "o3 was released in December 2024"
```

参照リストの各エントリに取得したスパン（verbatim 引用または要約）を記載する。
スパンは "—" の後に置く。これにより引用の根拠が検証可能になる。

## 言い換えデフォルト

原則: 言い換えで記述する。
直接引用（引用符 "" で囲む）は以下の場合のみ:
- 法的文言・契約・判決
- 定義の最初の導入（用語を定義している文そのもの）
- 個人発言の帰属（「〜と述べた」形式）

原文構造をなぞった言い換え禁止 — 完全に自分の言葉で書く。

## 矛盾ソースの処理

```
Source A は X と報告している [1]。Source B は Y と報告している [2]。
相違の原因: <特定できる場合は方法論/日付/スコープの違い、不明なら "不明"。>
```

どちらかを自動採用しない。例外: 片方が一次ソースで他方がその要約である場合のみ一次ソースを採用し、その旨を明記する。

## Limitations セクション

**全ての multi-source 回答の末尾に必ず付ける。**

```
**Limitations**
- <検索で Tier 1 ソースが得られなかったサブ質問>
- <未解決の矛盾>
- <ペイウォール・ログイン壁など、アクセスできなかったソース>
- <freshness 懸念（取得した最新記事の日付が古い場合）>
```

Limitations が空の場合でも「Limitations: なし」と書く（省略したのか本当になかったのかを区別するため）。

## Single-source マーキング

1 ソースのみが支持する主張には `[single-source]` を付ける。
読者がその主張の信頼性を判断できるようにするため。
