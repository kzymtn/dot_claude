## セキュリティ方針

### サプライチェーン攻撃対策

**基本方針**:

- 安全な方法でサードパーティライブラリをインストール
- バージョンロック機能を持つパッケージマネージャーを優先使用
- 適切な粒度で `git commit`

**Python**:

- pipenv の `Pipfile.lock` または uv の `uv.lock` でバージョン固定
- pyproject.toml の ``tool.uv]` セクションに `exclude-newer = "1 week"` と記述することによってリリース直後のパッケージをインストールすることを避ける。
- 信頼できないパッケージはインストールしない
  - するとしたらユーザーが手動でインストールする

**Node.js**:

- npm は使わず pnpm を使用する。
- pnpm の minimumReleaseAge 機能を使用して 最低でもリリースから1 週間以上が経過しているものを使用する。
- pnpm の `pnpm-lock.yaml` でバージョン固定

### 機密情報管理

- `.env` ファイルや `credentials.json` などの機密情報ファイルは**コミット禁止**
- ユーザーが明示的にコミットを要求した場合は警告を表示

### データプライバシー

- 個人データセット（`.csv`, `.pkl` 等）を扱う際は注意
- 大容量ファイル（`.tar.gz`, `.csv`, `.pkl`）は慎重に扱う
