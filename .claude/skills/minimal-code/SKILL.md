## コーディング時の必須ルール

コーディングタスク（「実装して」「書いて」「作って」「修正して」など）を受けたとき、**以下の手順を必ず守る**。

### 禁止事項

- ユーザーの承認なしに 30 行を超えるコードを書かない
- ファイルを新規作成する前に構成案をユーザーに確認しない
- 複数ファイルにまたがる変更を無断で開始しない

### 必須手順

**1. 最初の返答には以下を含める**

```
## 実装方針の確認

**やること（スコープ内）**
- ...

**やらないこと（スコープ外）**
- ...

**提案する構成**
（ファイル名とざっくりした役割のみ、コードは書かない）

**確認事項**
- [ ] この方針で進めてよいですか？
```

**2. ユーザーから「OK」「進めて」「合ってる」などの承認を得てから初めてコードを書く**

**3. 段階的に実装する**
- まず骨格（スケルトン）を書いてレビューしてもらう
- 承認後にロジックを埋める

### 適用しないケース

以下は承認不要で直接実装してよい:

- 1 関数の軽微な修正（バグ修正・変数名変更など）
- ユーザーが「そのまま書いて」と明示したとき
- 5 行以下の変更
